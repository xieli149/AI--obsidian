## 1. 标题风格

### 标题类型分布

黎子的标题有 **3 种主要类型**，各有公式：

**类型一：系列工具推荐型（最常用）**

> 公式：`运营必备AI工具推荐{序号}：{工具名}-{工具核心价值}`

示例：
- 「运营必备AI工具推荐10：腾讯元器-你的公众号好帮手」
- 「运营必备AI工具推荐9：得到get笔记，灵感记录神器」

**类型二：受众定位 + 教程型**

> 公式：`{目标人群}也能搞定：{工具名} {详细教程类型}`

示例：
- 「电脑白痴也能搞定：Claude Code 安装超详细图文教程」

**类型三：权威人物 + 反常识事实型**

> 公式：`{权威人物}的{内容主题}：{反常识/惊讶点}`

示例：
- 「Claude Code 之父的真实工作流：他不是一个 Claude，而是 10 个并行跑，太强了」

### 标题核心规律

| 规律 | 说明 |
|------|------|
| 系列感 | 「运营必备AI工具推荐」系列打造持续输出的人设 |
| 工具名必须出现 | 让读者一眼识别内容在讲什么工具 |
| 核心价值点睛 | 用一句话说清"这个工具解决什么问题" |
| 情绪词加持 | 「太强了」「好帮手」「神器」引发共鸣 |

## 2. 开头风格

### 常用开头方式：痛点场景引入

先用 **真实共鸣场景** 把读者拉进来，让读者看到自己。

**文章一（得到笔记）开头：**
> 我会经常遇到这些情况
> 👉 平时信息输入量巨大（对标的产品、灵感、文章、对话、课程……）
> 👉 灵感来得快但走得也快
> 👉 想做成个人知识库，却总是卡在"懒得整理"

→ 先列痛点，用 👉 符号逐条呈现，引起强烈共鸣。

**文章二（Claude Code）开头：**
> 最近龙虾🦞非常火，但是用过的都知道，龙虾还是比较烧钱的，做任务想要流畅需要付出挺多时间精力去调试，当然条件够的用龙虾🦞做长期业务是很不错的，如果你是刚开始接触AI，想用AI写点agent帮你处理日常一些琐碎工作，那可以先看看claudecode……

→ 先对比竞品，说明适用人群，自然过渡到主题。

**文章三（腾讯元器）开头：**
> 优势：腾讯生态产品使用丝滑
> 可以内置知识库，链接到公众号充当客服角色……

→ 直接讲工具优势，干货前置，不绕弯子。

### 开头常用句式

| 句式 | 示例 |
|------|------|
| 痛点罗列 | "我会经常遇到这些情况 👉 …… |
| 「坦白说」句式 | 坦白说——**越用越碎，越存越乱。** |
| 竞品对比引入 | "龙虾比较烧钱……如果是刚开始接触……可以先看看……" |
| 优势前置 | 直接列出工具的核心卖点 |

## 3. 行文结构

### 主要文章结构

**结构一：工具推荐文（最常用）**

```
开头痛点/场景 → 核心价值总述 → 功能逐条拆解 → 使用心得/注意事项 → 金句升华 → 作者介绍结尾
```

**结构二：教程文（步骤型）**

```
场景引入 → 工具介绍 → 分步骤教程（每个步骤附截图说明） → 注意事项 → 小结
```

**结构三：人物/干货解读文**

```
一句话总结核心内容 → 核心内容精要 → 分点解读（每点附「对小白理解」说明块） → 个人感悟 → 结尾
```

### 段落特点

- 每段控制在 3-5 行，适合手机阅读
- 每个观点一个段落，不在一个段落里堆叠多个信息
- 重要内容用 **加粗** 突出
- 使用分隔线 `---` 区分大章节

## 4. 用词习惯

### 核心用词风格

**口语化 + 真诚感**，不端着，不写"官方腔"。

| 用词特点 | 示例 |
|------|------|
| 第一人称「我」 | 「我是黎子……」「我以前用过……」「我现在开会只做一件事」 |
| 口语感叹词 | 「坦白说——」「嘿嘿」「真的好用到」「真的绝了」 |
| 简短有力句式 | 「按住录音。」「打开手机。说一句话。」 |
| 具体场景词 | 「灵感散」「任务杂」「信息乱」「素材多」「没时间整理」 |
| 行动导向词 | 「帮你搞定」「让你躺平」「把XX交给AI」 |
| 网络用语 | 「薅羊毛」「躺平」「打工」 |

### 语气特点

- 真诚坦率：不掩饰自己遇到过的问题（"坦白说——越用越碎，越存越乱"）
- 实用主义：每句话都指向"能用上"
- 轻松愉快：偶尔用 emoji（🦞🧠👆👉✨）

## 5. 语气语调

### 人设定位：陪伴型实战者

黎子的语气是 **「过来人的朋友」**，不是专家居高临下，也不是纯小白自言自语。

| 语气维度 | 表现 |
|------|------|
| 身份感 | 一个「正在用AI提效的运营人」，不是讲师 |
| 温度感 | 温和、有耐心，像朋友分享好东西 |
| 判断感 | 有明确态度（推荐/不推荐，适合/不适合），不中庸 |
| 边界感 | 「如果你也……」引入，不强制拉人入局 |

### 典型人设句

> 「我是黎子，一个拒绝在琐事中内耗的运营人。
> 专注分享【AI+运营/生活】实战心法，致力于把繁杂工作自动化。」

> 「如果你也想摆脱"执行工具人"的命运，欢迎入局。」

→ 不是「教你」，是「带你一起」。

## 6. 金句特点

### 金句位置

主要在每个功能模块的结尾，用于 **总结收束 + 升华价值**。

### 金句风格

**短小精悍 + 押韵感 + 画面感**

| 金句 | 出处 |
|------|------|
| 「越用越碎，越存越乱。」 | 得到笔记 |
| 「99%的灵感死在路上。」 | 得到笔记 |
| 「它让记录变轻、让整理变自动、让知识变可复用。」 | 得到笔记 |
| 「第一次用，会被震惊到的体验。」 | 得到笔记 |
| 「让AI为你打工。」 | 各篇文章结尾 |
| 「每天进步一点点，让AI为你打工。」 | 公众号签名 |

### 金句公式

**公式一：对比反差式**
> `{负面现象描述}。{改变后的正面结果}。`
> 例：「越用越碎，越存越乱。」「99%的灵感死在路上。」

**公式二：并列排比式**
> `{行动1}变轻、{行动2}变自动、{行动3}变可复用。`
> 例：「它让记录变轻、让整理变自动、让知识变可复用。」

## 7. 结尾风格

### 固定结尾结构（公众号文章标配）

```
个人介绍段落（1-2段）
→ 引导关注语
→ 账号名「运营人黎子的AI学习日记」
→ 「继续滑动看下一个 / 向上滑动看下一个」
```

### 个人介绍结尾模板

**工具推荐文结尾：**
> 我是**黎子**，一个拒绝在琐事中内耗的运营人。
> 专注分享【AI+运营/生活】实战心法，致力于把繁杂工作自动化。
> 如果你也想摆脱"执行工具人"的命运，欢迎入局。
> **关注我，让你的职业壁垒再高一寸**

**工具推荐文结尾（腾讯元器版）：**
> 大家可以直接在我公众号里面回复体验，嘿嘿，
> 后面我也会根据大家需求做提示词迭代调整
> **关注我，争取早日退休**

### 结尾核心逻辑

1. **用 1-2 句话定义自己**（运营人 + AI提效定位）
2. **一句话价值主张**（让AI为你打工 / 职业壁垒再高一寸）
3. **轻度引导关注**，不强硬，不刷屏

## 8. 排版习惯

### emoji 使用

| 使用场景 | emoji | 示例 |
|------|------|------|
| 列表项前缀 | 👉 | 「👉 平时信息输入量巨大」 |
| 强调点睛 | ✨ | 「✨ 它最强的三个能力：」 |
| 动物/心情 | 🦞🧠👍 | 「龙虾🦞」「脑子🧠」 |
| 主题标签 | 📌 | 「📌 今日要点」 |

### 分隔线

大量使用 `---` 分隔线区分大章节，增强阅读节奏感。

### 标题层级

- 一级标题（`#`）：文章主标题
- 二级标题（`##`）：功能模块标题（如「1）随手语音 → 自动变结构化内容」）
- 加粗（`** **`）：强调关键词和核心观点

### 列表样式

- 数字编号列表（分步骤教程）
- 👉 符号列表（痛点罗列）
- 普通 bullet（功能特点说明）

## 9. 其他写作特点

### 系列意识强

「运营必备AI工具推荐」系列是核心内容产品线，持续输出建立读者预期和订阅粘性。

### 教程即内容，内容即教程

不区分"这是教程"和"这是文章"，把每篇工具推荐都写成 **有温度的实操指南**，附上自己的使用场景和心得，不是冷冰冰的参数对比。

### 「对小白理解」说明块（干货解读文独创）

在解读外部干货（如 Claude Code 之父工作流）时，会在每个要点后加：

> **对小白理解：**
> 就像你同时和 5 个「远程实习生」聊天……

→ 这个机制让技术内容对运营读者完全可读，是**差异化护城河**。

### Prompt 作为内容

会把**自己写的提示词**直接分享到文章里（如腾讯元器的 prompt），让读者可以直接复制使用，增加文章实用价值。

### 结尾互动感

文章末尾常有「嘿嘿」「大家可以直接在我公众号里面回复体验」等引导读者实际操作的句子，把"看完文章"变成"立刻去试"。

## 后续写作参考规则

1. **标题必含工具名**，优先使用「运营必备AI工具推荐 N：」系列格式
2. **开头先用痛点/场景**，不要一上来讲定义
3. **每个功能点一篇文**，不贪多，把一个点讲透
4. **结尾必须有人设句**，「我是黎子……」固定出现
5. **金句放在每段结尾**，用短句，不要华丽辞藻
6. **加粗关键判断词**，让读者一眼扫到重点
7. **用「对小白理解」翻译复杂概念**，增加可读性
8. **emoji 适量**，用于强调和列表，不用过度装饰
