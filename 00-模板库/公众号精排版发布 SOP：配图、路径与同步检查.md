# 公众号精排版发布 SOP：配图、路径与同步检查

> 使用场景：文章已经有定稿，需要做成公众号后台可发布草稿。  
> 目标：避免标题识别错误、正文混入发布说明、图片不显示、图片不同步、水平分割线影响观感等问题。

## 一、核心原则

公众号同步前一定要拆成两类文件：

1. **发布包说明**：给作者自己看的，包含备选标题、封面建议、标签、发布时间建议。
2. **同步正文**：给 Wechatsync 用的，只包含正式标题、正文和图片。

不要直接同步「长文发布包」。

因为 Wechatsync 会把整个 Markdown 当成正文。如果里面有备选标题、封面建议、发布时间建议，这些都会进入公众号正文。

## 二、公众号同步正文的标准结构

同步文件第一行必须是正式文章标题：

```markdown
# AI 能操作电脑了，但你真的知道它在动哪里吗？
```

正文里只放：

- 正式标题
- 正文
- 正文内图片

正文里不要放：

- 备选标题
- 推荐标题说明
- 封面图设计建议
- 话题标签
- 发布时间建议
- 发布包目录说明

## 三、公众号精排版标准

同步前建议做一次精排版：

- 开头加一段简短导读
- 重点句加粗
- 长段落拆短
- 列举内容改成项目符号
- 密集概念改成表格
- 每个大章节之间适当插入配图
- 保留清晰小标题
- 不要全文过度加粗

## 四、删除文章里的水平分割线

公众号里 Markdown 水平分割线 `---` 会显示成横线，容易打断阅读节奏。

同步前删除所有单独成行的：

```markdown
---
```

可用规则：

```regex
^[ \t]*---[ \t]*$
```

删除后检查是否有过多空行。

## 五、图片生成 SOP

### 1. 不要让生图模型直接生成大量中文

很多生图模型会生成错字、乱码或奇怪文字。

更稳的方式是：

1. 用生图 API 生成无文字底图。
2. 自己用程序或设计工具叠加正确中文。
3. 最终导出 PNG。

### 2. 推荐生成 4 类图片

1. 封面图
2. 核心概念对比图
3. 风险边界图
4. 安全检查清单图

### 3. 当前可复用生图 API 参数

接口：

```bash
POST https://api.siliconflow.cn/v1/images/generations
```

模型：

```text
Tongyi-MAI/Z-Image-Turbo
```

建议参数：

```json
{
  "model": "Tongyi-MAI/Z-Image-Turbo",
  "image_size": "1024x1024",
  "batch_size": 1,
  "num_inference_steps": 20,
  "guidance_scale": 7.5
}
```

注意：

- API Key 不要写进笔记、脚本或 Skill。
- 每次调用时使用环境变量。

## 六、图片路径规则

这是最容易出错的地方。

### 1. Obsidian 预览用

Obsidian 里不要直接写：

```markdown
![图](/Users/lizi/Desktop/xxx/image.png)
```

这样 Obsidian 会把它当成库内路径，可能提示找不到。

Obsidian 预览应该写：

```markdown
![图](file:///Users/lizi/Desktop/xxx/image.png)
```

如果路径里有中文或空格，需要 URL 编码。

### 2. Wechatsync 同步用

Wechatsync 同步时不要用 `file://`。

同步用 Markdown 应该使用相对路径：

```markdown
![图](images-final/cover-final.png)
```

前提：

- Markdown 文件在 `publish-packs/` 目录
- 图片在 `publish-packs/images-final/` 目录

### 3. 必须保留两套 Markdown

建议每篇文章都生成：

```text
公众号精排版正文-含图片-预览用.md
公众号精排版正文-含图片-同步用.md
```

预览用：给 Obsidian 看。  
同步用：给 Wechatsync 上传公众号草稿。

## 七、同步前检查清单

同步前逐项检查：

- 第一行 H1 是正式文章标题
- 正文里没有备选标题和发布说明
- 没有单独成行的 `---`
- 图片文件真实存在
- 预览用文件使用 `file://`
- 同步用文件使用相对路径
- 同步用文件里没有 `/Users/...` 绝对路径
- 同步用文件里没有 `file://`
- Wechatsync 能连接 Chrome Extension
- 公众号平台显示已登录

## 八、Wechatsync 同步命令

```bash
WECHATSYNC_TOKEN="$WECHATSYNC_TOKEN" wechatsync sync "公众号精排版正文-含图片-同步用.md" -p weixin
```

同步成功时，终端应该看到：

- 标题识别正确
- 发现本地图片
- 图片上传成功
- 草稿同步成功

## 九、同步后检查

进入公众号后台草稿箱，检查：

- 标题是否正确
- 图片是否全部显示
- 表格在手机端是否可读
- 重点加粗是否适量
- 是否有多余发布说明
- 是否有旧草稿需要删除
- 封面图、摘要是否需要手动补充

## 十、下次直接使用的指令

可以直接对 AI 说：

```text
请按「公众号精排版发布 SOP」处理这篇定稿：生成配图、做公众号精排版、删除所有水平分割线，生成 Obsidian 预览用和 Wechatsync 同步用两套 Markdown，最后同步到公众号草稿箱前先让我确认。
```
