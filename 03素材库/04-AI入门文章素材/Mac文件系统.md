**亦仁引言：**  
最近星球里越来越多人把主力办公机换成了 Mac。原因很直接：Codex、Claude、飞书 CLI 这些 AI 工具在 Mac 上体验更好。

但换过来之后，比"怎么用 AI"更先卡住人的，是一个更基础的问题——Mac 的文件系统搞不清楚。

文件夹之间什么关系？iCloud 到底做了什么？为什么一个叫 Codex 的文件夹出现在两个地方？Finder 里看到的和应用弹窗里看到的对不上？

我自己也被这些问题困扰过。有次发现家目录下同时有一个 `codex` 和一个 `.codex` ，差点把配置删了。后来花时间把 Mac 的文件逻辑从头理了一遍，发现结构本身不复杂——只是没人系统讲过。

这篇文章做的就是这件事。不讲技巧，讲结构。给你一张地图，让你知道东西在哪、为什么在那。地图有了，剩下的遇到再查就好。

---

很多人用了 Mac 好几个月，始终搞不清一件事：我的文件到底在哪？

装了 Claude，发现多出一个 claude 文件夹。用了 Codex，又冒出一个 Codex 文件夹——有时在这里看到，有时在那里看到，有时又找不到。打开 Finder 看到的和别的应用弹窗看到的不一样。iCloud 好像在背后做了什么事，但不知道做了什么。

这些困惑的根源都一样：你还没有一张 Mac 文件系统的地图。这篇文章帮你建起来。

## 一、家目录：所有东西的起点

Mac 上每个用户都有一个"家目录"，路径是 `/Users/你的用户名` 。你在这台电脑上的一切——桌面上的文件、下载的东西、装的应用的数据、AI 工具生成的项目——几乎都住在这个目录下面。

打开家目录，你会看到这样一堆文件夹并排放着：

```
/Users/你的用户名/
├── Desktop/          ← 桌面，就是你屏幕上看到的那个桌面
├── Documents/        ← 文稿，系统建议放文档的地方
├── Downloads/        ← 下载，浏览器下载的文件默认在这里
├── Applications/     ← 部分应用会安装到这里（系统级应用在另一个 Applications）
├── Pictures/         ← 图片，照片图库也在这里
├── Library/          ← 资源库（默认隐藏），应用的配置和缓存住在这里
│
│   ← 以上是系统自带的。以下是你自己创建的、或者应用自动创建的：
│
├── claude/           ← Claude 生成的项目文件夹
├── codex/            ← Codex 生成的项目文件夹
├── Projects/         ← 你自己建的工作目录
└── ...
```

**第一个关键认知：这些文件夹全部是平级的。** Desktop、Documents、Downloads、claude、codex——它们都是家目录的直接下级，彼此之间是并排关系，不是互相嵌套。

这一点之所以重要，是因为大多数人平时不看家目录全貌，只通过 Finder 侧边栏进"文稿"或"下载"。一旦有应用在家目录下创建了新文件夹（比如 AI 工具生成的 claude、codex），你可能完全看不到它们——直到在某个弹窗里突然冒出来，一脸困惑。

## 二、Finder：你每天用的文件管理器，但它只让你看到局部

Finder 是 Mac 上管理文件的核心工具，但它有一个特点： **默认只给你看一个精简视图，不是全貌。** 很多困惑都出在这里。

### 侧边栏：它是快捷入口，不是完整地图

Finder 左侧有一列常驻的入口，叫侧边栏。默认状态下，你会看到：

**"个人收藏"栏** ：桌面、文稿、下载、AirDrop 等。这些是系统帮你设好的快捷方式，点一下就能跳到对应的文件夹里。

**"iCloud"栏** （如果你登录了 iCloud）：iCloud Drive，以及开启了同步的桌面与文稿。

**"位置"栏** ：你的 Mac 硬盘、外接磁盘、网络位置等。

侧边栏最容易造成困惑的一点是： **当你点击侧边栏里的"文稿"，Finder 会直接跳进 Documents 文件夹的内部，只展示里面的内容。** 你看不到和 Documents 平级的其他文件夹——Desktop、Downloads、claude、codex 这些邻居全部被挡在视野之外。

这就是为什么你在 Finder 里点"文稿"看到的内容，和某个应用弹出的文件选择窗口里看到的内容对不上：应用的弹窗可能打开了家目录本身，你能看到所有文件夹并排；而 Finder 侧边栏把你送进了 Documents 内部，视野被收窄了。

### 怎么看到家目录的全貌

**方法一：快捷键。** 在 Finder 里按 `Cmd + Shift + H` ，直接跳到家目录，看到所有文件夹平铺。

**方法二：把家目录加到侧边栏。** 用上面的快捷键进入家目录后，把窗口顶部显示的文件夹图标（带你用户名的那个小房子图标）直接拖到侧边栏的"个人收藏"区域。以后一点就能回到全貌视图。

**方法三：在侧边栏显示更多位置。** 打开 Finder → 菜单栏"设置"（或 `Cmd + ,`）→"边栏"标签页，可以勾选要在侧边栏显示的内容。建议把你的用户名（家目录）勾上。

### 开启路径栏——最被低估的功能

Finder → 菜单栏 → 显示 → 显示路径栏（或按 `Option + Cmd + P` ）。

开启后，Finder 窗口底部会出现一条路径，清楚显示你当前所在位置的完整层级。比如你进入了一个 Codex 项目文件夹，路径栏会显示：

```
你的用户名 > Documents > Codex > 2026-06-02 > my-project
```

一眼就能看出：这个项目在 Documents 里面，不是在家目录下。遇到"这个文件夹到底在哪"的困惑时，第一件事就是看路径栏。

路径栏还有一个用法： **双击路径上的任何一个层级，可以直接跳到那一层。** 比如双击"你的用户名"，就直接回到家目录。

### 切换视图：分栏视图最适合理解层级

Finder 有四种视图模式，通过窗口顶部的图标或快捷键切换：

- 图标视图（ `Cmd + 1` ）：文件摆成网格，适合看图片，但看不出层级关系。
- 列表视图（ `Cmd + 2` ）：按名称排列，可以展开子文件夹，适合浏览大量文件。
- **分栏视图（** `Cmd + 3` \*\*）\*\*：最推荐。每点进一层文件夹，右边就多出一栏，左边的上级始终可见。你能同时看到文件在哪一层、它的上级是谁、它的邻居有哪些。
- 画廊视图（ `Cmd + 4` ）：大预览图，适合筛选图片。

建议日常用分栏视图。它能让你始终保持对"我在哪一层"的感知，不容易迷路。

### 隐藏文件：有些文件夹你看不到，但它们一直在

Mac 会隐藏以点（.）开头的文件和文件夹。在 Finder 里按 `Cmd + Shift + .`（英文句号），可以切换显示/隐藏。

这跟 AI 工具有什么关系？当你安装了 Claude Code、Codex 这类工具后，它们会在家目录下创建以点开头的配置文件夹（比如 `.claude` 、`.codex` ），用来存放登录信息和设置。这些文件夹和你看到的 `claude` 、 `codex` 文件夹不是一回事——带点的存的是工具的配置，不带点的存的是你的项目文件。后面"应用存的和你存的"那部分会详细讲。

### 快速跳转到指定位置

在 Finder 里按 `Cmd + Shift + G` ，会弹出一个"前往文件夹"的输入框。直接输入路径（比如 `~/Documents` 或 `~/Projects` ），回车即可跳过去。当你知道确切路径但不想一层层点过去时，这是最快的方式。 `~` 是家目录的简写。

## 三、你的文件到底在哪：理解"应用存的"和"你存的"

Mac 上有一条重要的规则： **应用本身、应用的配置、你的文件，这三者是分开存放的。**

**应用本身** 住在"应用程序"（Applications）文件夹里。你从 App Store 装的、从官网下载拖进去的，都在这里。比如 Claude 桌面应用、飞书、Obsidian。

**应用的配置和数据** 住在家目录下的"资源库"（Library）文件夹里，以及以点开头的隐藏配置文件夹里。你不需要日常管理这些，但要知道它们的存在。

**你的文件** ——你创建的文档、AI 工具为你生成的项目、你下载的素材——住在你自己选择的位置，或者应用替你选择的位置。

这三者互相独立。 **删掉你的项目文件夹不会影响应用本身；卸载应用也不会删掉你的项目文件。** 理解这一点，"删了会不会出事"的焦虑就消失了一大半。

### AI 工具创建的文件夹在哪

不同的 AI 工具存放项目文件的方式不完全一样，但有一个共同点：它们通常会在某个位置为你创建一个以工具名命名的文件夹。

比如你用 Claude 处理文件时，它可能在家目录下创建一个 `claude` 文件夹来放生成的内容。Codex 可能在你当前打开的位置创建一个 `Codex` 文件夹。如果你在不同的地方分别使用过这些工具，就可能出现同名文件夹散落在多个位置的情况：

```
~/codex/                     ← 在家目录下创建的
~/Documents/Codex/           ← 在文稿里创建的
```

这两个不是同一个文件夹的两种显示——它们就是两个独立的文件夹，只是碰巧名字相似。内容可能不同，创建时间也不同。

怎么判断到底有几个、分别在哪？打开 Finder，按 `Cmd + Shift + H` 回到家目录，切换到分栏视图，逐层查看。配合路径栏，就能看清每个文件夹的真实位置。

### 带点的和不带点的：两种容易混淆的文件夹

在家目录下，你可能会同时看到：

```
~/claude/          ← 不带点，普通文件夹，存的是你的项目文件
~/.claude/         ← 带点，隐藏文件夹，存的是工具的配置和登录信息
```

这是两个完全不同的东西。不带点的那个是你的工作产出，可以自由移动、重命名、整理。带点的那个是工具的内部数据，不要随意动——动了可能需要重新登录或重新配置。

如果你在 Finder 里看不到带点的文件夹，按 `Cmd + Shift + .` 显示隐藏文件即可。

## 四、iCloud：什么被同步了，什么没有

iCloud 是很多人最困惑的部分。核心问题只有一个：我的文件是在本地，还是在云上？答案取决于你开了哪些设置。

### iCloud Drive

开启后，Finder 侧边栏会出现"iCloud Drive"入口。你主动放进去的文件会同步到云端，并出现在你所有登录了同一个 Apple ID 的设备上。这部分比较好理解。

### "桌面与文稿文件夹"同步——最容易造成困惑的开关

在系统设置 → Apple ID → iCloud → iCloud Drive 里，有一个"同步此 Mac 的桌面与文稿文件夹"选项。

开了之后，你的"桌面"（Desktop）和"文稿"（Documents）两个文件夹就被 iCloud 接管了。它们的内容会同步到云端，在 Finder 侧边栏里也会从"个人收藏"移到"iCloud"那一栏下面。

**对普通使用来说，这是个好功能。** 你在这台 Mac 桌面上放的文件，另一台 Mac 自动就有。

**但如果你的 Documents 里有 AI 工具创建的项目文件夹，问题就来了。** 代码项目通常包含大量小文件，iCloud 会逐一尝试同步，导致极其缓慢甚至报错。所以，AI 项目文件夹建议不放在 Desktop 或 Documents 里（具体怎么放，第五部分会讲）。

### "优化 Mac 储存空间"——另一个容易踩的坑

这个开关在 iCloud Drive 设置里。开了之后，当本地空间不足时，系统会自动把不常用的文件从本地移除，只在云端保留。本地只留一个占位符——你在 Finder 里还能看到文件名，但文件本体不在你的电脑上了。

这些被"优化"掉的文件旁边会有一个云朵下载图标。双击才能重新下载到本地。如果没有网络，就打不开。

如果你不知道这个机制，就会遇到一个非常困惑的状况：项目文件明明在 Finder 里看得到，但 AI 工具运行时报"文件不存在"。原因就是：文件已经被替换成了云端占位符，只有名字还留在本地。

### 多台设备之间，什么跟着走、什么不跟着走

**会通过 iCloud 同步的：** 桌面和文稿的内容（如果开了同步）、iCloud Drive 里的文件、照片（如果开了 iCloud 照片）、Safari 书签、备忘录、提醒事项、日历、通讯录、钥匙串里存储的密码。

**不会同步的：** 应用本身（每台 Mac 需要单独安装）、Downloads 文件夹的内容、你在家目录下自己建的文件夹（比如 ~~/Projects，除非你主动放进 iCloud Drive）、大多数应用的配置和登录状态、AI 工具的隐藏配置文件夹（~~ /.claude、~/.codex 等）。

一个常见的误解是"我两台 Mac 登了同一个 Apple ID，应该什么都一样"。不是的。iCloud 同步的是特定文件夹里的内容，不是"整台电脑的所有东西"。应用要每台机器自己装，AI 工具的登录要每台机器自己走一遍。

### iCloud 存储空间

每个 Apple ID 有 5GB 免费 iCloud 空间，很容易满。满了之后同步停止，备份也做不了。

查看占用：系统设置 → Apple ID → iCloud → 管理储存空间。通常最占空间的是照片和 iPhone/iPad 备份。

## 五、怎么组织你的文件夹

理解了前面四部分，这一步就是把认知落成习惯。

### 给 AI 项目一个固定的家

在家目录下建一个文件夹，专门放所有 AI 相关的项目。叫 `Projects` 或 `Work` 都行：

```
~/Projects/
├── my-app/             ← 用 Claude 做的一个应用
├── blog-site/          ← 用 Codex 搭的网站
├── data-analysis/      ← 一个数据分析项目
└── ...
```

**为什么放在家目录下，而不是放在"文稿"里？** 因为如果你开了"桌面与文稿同步"，Documents 里的内容会被 iCloud 接管。AI 项目文件夹里大量小文件会让 iCloud 同步出问题。 `~/Projects` 在 Documents 之外，不受 iCloud 影响。

建好之后，把这个文件夹拖到 Finder 侧边栏的"个人收藏"区域，以后一点就能进去。

### 整理已经散落的文件夹

如果你已经有多个同名或近似的文件夹散落在不同位置（比如家目录下有个 codex，Documents 里也有个 Codex），可以这样处理：

1. 按 `Cmd + Shift + H` 回到家目录，配合分栏视图和路径栏，先搞清楚每个文件夹的确切位置和里面的内容。
2. 确认哪个是在用的、哪个是旧的。
3. 不确定能不能删的，先改名加个 `-old` 后缀（比如 `codex-old` ），正常使用几天。
4. 没问题后，把需要保留的内容迁移到 `~/Projects` 下的对应位置，再处理旧文件夹。

注意：只整理你自己的项目文件夹（不带点的），不要动以点开头的隐藏配置文件夹（如 `.claude` 、`.codex` ）。后者存的是工具的登录信息和设置，动了会需要重新配置。

### 养成"从固定位置开始"的习惯

AI 工具创建文件夹时，通常会放在你"当前所在的位置"。如果你每次从不同的地方开始（有时在桌面，有时在文稿，有时在家目录），就会在不同的地方各留下一个文件夹。

保持一个简单的习惯：每次开始新项目，先进入 `~/Projects` ，再创建项目文件夹。散乱不会自动消失，但这个习惯能防止它继续生长。

### 多台 Mac 之间同步项目

你自己建的 `~/Projects` 不走 iCloud（这是我们想要的）。那多台 Mac 之间怎么同步项目？

**推荐用 Git/GitHub。** 在 A 电脑完成工作后推送到 GitHub，在 B 电脑拉取下来。这是行业通用的做法，不会产生 iCloud 那样的同步冲突。

如果你不熟悉 Git，另一个临时的办法是把需要同步的项目文件夹手动放进 iCloud Drive（注意是 iCloud Drive，不是 Documents）。但长期来看，代码类项目还是建议用 Git。

### 一些有用的 Finder 习惯

**随时看路径栏。** 遇到"这个文件夹在哪"的困惑时，看一眼窗口底部。

**用** `Cmd + Shift + H` \*\*回到家目录。\*\* 当你被 Finder 的层层跳转搞晕时，这是"回到起点"的快捷键。

**用分栏视图（** `Cmd + 3` \*\*）浏览文件。\*\* 它能让你同时看到文件所在的层级和它的邻居，不容易丢失方向感。

**把常用位置加到侧边栏。** 把 `~/Projects` （或你的工作目录）拖进 Finder 侧边栏，和"文稿""下载"并排。以后一点就到。

## 最后，记住一张地图

整篇文章讲的其实就是一件事：Mac 的文件系统是一棵树，家目录是树根，所有东西从这里长出来。Finder 侧边栏是几根标记过的树枝，方便你快速够到常用的位置——但它不是整棵树。

当你搞不清一个文件夹在哪、和别的文件夹是什么关系时，回到家目录看全貌。路径栏告诉你"你在第几层"，分栏视图告诉你"你的邻居是谁"。

不需要记住所有细节。你需要的只是这张地图——知道怎么回到起点、怎么看清当前位置、怎么辨别方向。剩下的，遇到了再查就好。
