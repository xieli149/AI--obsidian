看完本文，你会搞明白：

•

为什么你下载的那些高 star Skills，大多数到自己手里都跑不通

•

真正有效的 Skills 长什么样、是怎么来的

•

99% 的人在做 Skills 之前都跳过的关键一步：给 AI 写一份「我是谁」

•

我的小红书虚拟电商 Skills 体系是怎么搭起来的

•

我的公众号 Obsidian + Claude Code 系统怎么跑起来的

•

怎么从零原创一个真正属于你自己的 Skill

大家好，我是简哥。

我是一位视觉创意设计师，在一家普通的民营公司上班，主要负责电商、市场宣传方面的设计工作。

下班时就是我个人的主场，拿起小红书电商项目就是干，从 6 点做到 12 点，一天 14-15 个小时，天天如此！

励志要用 AI + 审美做一家一人公司。

这是我第七次参加大航海了，上过好多条船，也去过不同的海域，领略过不一样的风景，最后还是选择了"电商号"扬帆起航，驶往深海寻鲸。航行的旅程很快乐，也很焦虑，也许这就是旅行的意义。一抬头即将靠岸，这一路的所见所得，我想做一个简单的分享，也是对自己旅程的一个记录📝

加入生财这一年多，前前后后也做过其他几个项目，小红书实物电商、AI 自媒体等，都没有取得实质性的结果。

后面看了大量的精华帖，禅定了好几天，最后选择了小红书虚拟电商赛道，慢慢的就做到了现在。

原本以为实物电商是跟自己最匹配的项目，因为之前有做淘宝、抖音电商的经验，所以一开始就选择了小红书实物电商。

但是高频次的外出拍照、不断的选品、不断的分类大量的手机照片 live 图，感觉每周都在做同样却不见正反馈的事情，而且发货、售后、退款一度搞的我店铺瘫痪，心力交瘁。

不是说不能做，是虚拟电商可能更加的省力、省事，后端不用这么操心，做好前端就行。

对于虚拟电商，我要做的就是静下心来潜心创作就行——不用发货、不用售后、不用外出、不用拍照、不用预约场地、不用每周整理 700-800 张的实拍图，我觉得非常省心了！

但虚拟电商虽然说省心，"潜心创作"这件事本身又把我难住了。

每天要在 4 个账号上发笔记，一个人盯着是不行的，必须借 AI 的力。所以从去年开始我就一直在用 Claude，从聊天窗口聊到 Claude Code，前前后后跑了好多版本的工作流。

刚开始用 AI 那段时间，我就是个标准的"拿来主义者"。

到处去看高手分享的 Skill 包，看哪个 star 高就收藏下载，看到圈友群里有人发 Skill 就先存起来。觉得自己手上 Skill 越多，AI 干起活儿来肯定越厉害。

结果用着用着就发现不对劲。

那些下载来的 Skill，大部分都跑不动我的业务。要么任务场景完全对不上，人家是给程序员写代码用的，我是做小红书内容的；要么勉强跑通了，输出的东西怎么看怎么别扭，根本不是我想要的味道，一股浓浓的 AI 感，怎么改都改不掉。这边出问题，那边出问题，越改越糟糕，越改越不行。

那段时间我特别困惑，到底是这些 Skill 不行，还是我自己不会用？

直到后面我才慢慢想明白，问题根本不在 Skill 上。

问题在于，AI 根本不认识我。它不知道我是谁、是做什么的、审美在哪个水平、服务的是哪群人、以前踩过哪些坑、表达习惯是什么样的。一个不认识我的 AI，用一套不是为我写的 Skill，去干我的活儿，凭什么能跑通呢？

![](https://sphere-search-mobile.oss-cn-shanghai.aliyuncs.com/upload/doc/blocks/Cz8vbMwYioYRuZxVbj0cfrQXnEb?x-oss-process=image/auto-orient,1/resize,m_lfit,w_1280,h_16000/quality,q_75/format,webp)

这件事我想了很久，也走了不少弯路。这篇文章想把我趟过的这些坑分享出来：Skills 是什么、为什么别人的 Skills 到你手里就废、真正有效的 Skills 长什么样，以及最关键的那一步，在搭 Skills 之前，你得先把自己这个人完整地交给 AI。

聊到这儿，得先把 Skills 这件事讲清楚。

我发现圈子里很多人，包括我自己刚接触的时候，对 Skills 都有一个误解，以为它是个很高大上的技术活儿，是程序员才能搞的事情，是写一堆代码堆出来的东西。

其实根本不是。

说人话，Skills 就是你赋予 AI 去执行某项具体任务的能力模块。这是字面意思，但这么说还是有点抽象，我换个方式说。

你把 AI 想象成一个新来的实习生。这个实习生很聪明，悟性高、学东西快、24 小时不喊累。但聪明归聪明，他刚到你公司，对你的业务一无所知，对你的工作方式也不了解。你让他直接干活，他只能凭聪明硬猜，猜出来的东西十有八九不是你想要的。

那怎么办？你得给他写个"岗位说明书"。

告诉他这个岗位是干嘛的、每天要做什么、做的时候按什么标准、输出的东西放到哪里、什么情况下找谁汇报。这份说明书写得越细，他干起来就越接近你想要的样子。

我预测，它后面就能拥有你 70% 以上的功力，等于是你的一个数字分身。

Skills 本质上就是这份「岗位说明书」，只不过我们写给 AI 的版本，更具体、更结构化、更可执行。

比如我做小红书内容，每天都要写笔记。这件事高频、重复、有套路，我就把"怎么写一篇 Leo's Design Notes 的小红书笔记"这件事封装成了一个 Skill。从选题、到文案、到关键词布局、到评论区文案，全部按我自己的标准写进去。下次要写新笔记的时候，AI 直接读这个 Skill 按规则走，输出的东西就有我的味道。

所以 Skills 的核心特点是这两个字，重复。

如果你有一件事每周、每天都要做，且做的方式有自己的章法，那这件事就值得做成 Skill。反过来说，如果一件事你只做一次、或者每次做的方式都不一样、或者你自己都没想清楚怎么做，那它就不适合做成 Skill。这种情况下你直接跟 AI 对话让它一次性解决就行，没必要费劲做 Skill。

我打几个具体的比方：

•

写小红书笔记：每天都写、有固定的写法和标准，Skill 化之后效率翻倍，绝对值得做

•

写商务合同：可以做，但要看频次，一个月写一两份的，做 Skill 不如直接对话

•

写公众号长文：值得做，我每周写、有自己的结构、节奏、反 AI 味规则，封成 Skill 之后稳定输出

•

帮孩子改作业：算了，每次的题目都不一样，没有重复性，做不出 Skill

理解到这一层，你就知道 Skills 不是越多越好。真正有效的 Skills，是少而精地包住你最高频的那几件事，让 AI 在这几件事上稳定地输出你想要的东西。

我现在 Obsidian 里跑的 Skills 大概十几个，覆盖了我业务里所有的高频任务。每多一个能跑通的 Skill，意味着我每天要做的事情就少一件，时间就被释放出一块。这个账算下来很划算。

那问题就来了。既然 Skills 的本质是"岗位说明书"，为什么我之前下载了那么多别人写好的"说明书"，到自己手里都不好使？

回到那个困扰我很久的问题。

我曾经下载、收藏过的 Skills 不下几十个，真正能跑通我业务的，一个都没有。一开始我以为是自己技术不行、配置不对，后来才慢慢看清楚，这件事跟技术没关系。

别人的 Skills 到我手里跑不通，原因就两条。

4.1 第一条：任务场景根本对不上

我观察了很久 GitHub 上那些高 star 的 Skills，发现一个规律：绝大部分 Skills 解决的，都不是我急需要解决的那个任务。

要么是给程序员写代码用的，要么是做翻译、做总结的，要么是市面上已经有大量成熟工具的领域。

随便举三个目前重复造轮子最严重的方向：PPT 制作、小红书封面图生成、公众号文章排版。

这三类 Skills 在网上一搜一大把，看上去都做得挺好。但你想想，Gamma 把 PPT 这件事卷成什么样了？Canva、即时设计、稿定设计把封面图工具做到了什么程度？秀米、壹伴这些公众号排版工具是不是已经无敌了？用现成工具，比你自己去做一个 Skill 高效十倍不止。

真正有价值的 Skill，是解决别人没解决、或者没你解决得好的那个具体场景。 比如我这个赛道，没有任何一个现成工具能帮我搞定"Leo's Design Notes 风格的小红书笔记 6 张图文 + 一篇正文 + 关键词布局 + 评论区文案"这一整套东西。这种活儿只能自己搭，搭出来就是壁垒。

但市面上的 Skill 包基本不会做这么垂直的事儿，做了也没人下载。所以你看到的 Skills，绝大多数是通用化的、面向最大公约数的。这些东西到了你具体的业务场景里，基本上就是一件不合身的衣服，套上去哪儿都别扭。

4.2 第二条：缺少你的隐性知识密度

这一条更深，也是大多数人想不通的地方。

就算我们退一步，假设你找到一个任务场景跟你完全匹配的 Skill，比如就是写小红书笔记的，下载下来用，照样跑不出你想要的东西。为什么？

因为 Skills 的真正价值，从来不是它的代码结构有多精妙、Prompt 写得有多漂亮。它的价值，来自于背后封装了多少「只有你才懂」的东西。 这些东西我把它叫做"隐性知识"。

举个例子，做我这个产品的小红书笔记，标题怎么写、痛点怎么戳、知识点怎么排、钩子放在哪一句、配图哪一张放在 P1 哪一张放 P5、评论区第一条说什么第二条说什么，每一个动作背后，都是我做了 567 篇笔记踩出来的经验。

我看到一个标题，3 秒钟就知道这玩意儿能不能爆。但你让我说出来"为什么"，我可能要费半天劲才能给你讲清楚。这就是隐性知识，它在我脑子里，是直觉，是手感，是"看一眼就知道行不行"的判断力。

别人的 Skill 里没有这些东西，它有的只是一个通用的骨架，一个写小红书笔记的标准模板。把这个骨架拿到我业务里来用，跑出来的东西就是网上随处可见的那种"看上去对、用起来废"的内容。所以你业务上的 skills 一定要是你亲身经历的，是一步步犯过错、修改过、提炼出来的。这个没有办法一步到位，直接把别人的成品套过来用。

我做的产品定位是中高级视觉设计师，他们要的是高级感、是审美突破、是"被甲方说太土"那种具体痛点的解药。这些东西我自己是有手感的，我能精准地戳到。但一个通用 Skill 不知道这些，它写出来的东西可能也能叫"小红书笔记"，但跟我想要的根本不在一个层次。

结构可以复制，隐性知识没法复制。 这是别人的 Skills 到你手里就废的根本原因。

4.3 我自己踩坑后想明白的事

那段时间我反复在想这件事，到底什么样的 Skill 才是有效的？后来逼着自己回答两个问题：

第一个问题：这个 Skill 解决的任务，市面上有没有现成工具？如果有，且做得比我好，我就不该花时间做 Skill，直接用工具。

第二个问题：这个 Skill 里封装的判断标准，是不是我自己业务里独有的、别人复制不走的？如果是通用规则，那这个 Skill 价值不高。

把这两个问题想通了以后，我才意识到一件事：真正属于我的 Skill，不可能从外面找来。

它必须从我自己的业务里长出来，从我每天反复在做的高频任务里长出来，从我自己的判断标准和审美体系里长出来。也就是说，Skills 这件事的本质，根本不是技术问题，是认知问题，是你有没有把自己业务里那些只有你才懂的东西，沉淀下来、结构化下来、交给 AI 的问题。

而要做到这一点，有一个所有人都跳过的关键前提。

前面说了那么多，现在到了这篇文章我最想讲的部分。

如果你问我，用了这么久 AI，我悟到的最重要的一件事是什么，我会告诉你四个字：先让 AI 认识你。

这件事听起来特别简单，简单到大部分人根本不会当回事。大家一上来就是研究 Prompt 怎么写、研究哪个 Skill 好用、研究怎么搭工作流，没有一个人停下来问问自己：AI 知道我是谁吗？

它不知道。

你每开一个新对话，对面那个 AI 就是一张白纸。它不知道你做什么、你的水平在哪、你服务谁、你的脾气和审美是什么样的。你不告诉它，它就只能按一个"通用人类"的方式来揣测你，揣测出来的东西，当然就是网上那种最大公约数的味道。

我是怎么发现这件事的呢？

其实在做 Leo's Design Notes 这个产品之前，我就已经摸到了一点门道。当时我选品的方式跟大部分圈友都不一样，别人是正向选品，看数据、找对标、看哪个火，我是反过来的，先搞清楚"我是谁、我能赚谁的钱"。

为了搞清楚这件事，我跟 Claude 聊了无数个日日夜夜。我们从最根本的地方开始聊，把我这个人一层一层扒开：

•

我有哪些技能和技术？我擅长什么？我的兴趣在哪？

•

我过去经历过什么？我的背景是什么样的？

我把这些全部写成了一篇文章发给 Claude，让它去判断，我到底是个什么样的人，适合做什么样的事。

当时只是为了选品，但后来我才反应过来，那篇"我是谁"的文章，才是后面一切的地基。

正是因为 AI 足够了解我，它才能帮我从设计师这个大群体里，一层层往下切，切到视觉设计师，再切到 3-5 年的中高级视觉设计师，再切到"被甲方说太土、做不出高级感"这个最具体的痛点。换成一个不认识我的 AI，它给我的建议只会是"做个设计教程吧""教 PS 吧"这种谁都能想到的答案。所以“懂你”这一词的份量有多重了，你细品一下。

后来搭 Obsidian + Claude Code / Codex系统的时候，我把这件事彻底固定了下来。

我在系统里建了一个文件，专门写"我是谁"。它有个学名叫 [CLAUDE.md](http://claude.md/) / [Agants.md](http://agants.md/) ，你也可以理解成我的个人档案。这个文件 AI 每次启动都会自动读一遍，读完它就"认识"我了，知道在为一个什么样的人干活。

这份档案我写得很细，包括我的基本信息、性格、核心能力、价值观，最关键的是，我把我这 10 年的真实经历线全写进去了。 简化一下大概长这样：

简哥 · 人物档案（节选）

一句话定位：用 AI 系统化审美认知的超级个体。

核心能力：

•

审美判断力（天赋型）：15 年视觉设计实战，深度拆解过 25 个全球顶尖品牌的设计体系，一眼能看出什么是好的、什么是差的

•

第一性原理思维：不接受"因为别人都这么做"，喜欢穿透表象找本质

•

AI 内容系统化：用 Claude 从零手搓了一套审美认知知识库产品

人生经历线：

•

做设计做到公司最高工资，觉得天花板太低，裸辞去做房产销售，一年没开一单

•

回归设计进阶到管理层，公司拿了 4000 万融资，又因政策黄了，团队解散我被裁

•

第一次创业抖音、第二次微信小程序、第三次淘宝，全失败

•

离开打拼十几年的上海，回老家继续折腾

•

抖音男装、抖音财经、抖音电商，又是接连失败

•

焦虑到每天下班在草地上来回徘徊，想不出能做什么

•

加入生财，第 8 次创业做小红书虚拟电商，终于跑通

核心信念：

•

不选赛道，选人群。你的痛点，就是一群人的痛点。

•

方向不是想出来的，是在对的圈子里、用对的方法论、做匹配自己的事情，慢慢"长"出来的。

![](https://sphere-search-mobile.oss-cn-shanghai.aliyuncs.com/upload/doc/blocks/PWFTbqDCko4JtAxFVqXcCQFQnEe?x-oss-process=image/auto-orient,1/resize,m_lfit,w_1280,h_16000/quality,q_75/format,webp)

你看，这份东西里没有一句是技术，全是"我这个人"。但恰恰是这份东西，决定了我后面所有 Skill 的质量。

因为 AI 读完这份档案，它就不再是那个揣测"通用人类"的 AI 了，它成了一个"认识简哥"的 AI。它知道我是设计出身、知道我审美在线、知道我服务的是中高级设计师、知道我说话直接不装带点江湖气、甚至知道我失败过 7 次所以特别能共情那些还没跑通的人。这时候我再让它帮我写笔记、做选题、改文案，它写出来的东西，才有我的味道。

我用一个比喻你就懂了。

Skills 是种子， [CLAUDE.md](http://claude.md/) 是土壤。

你从别人那儿挖来一颗再好的种子，种在一片贫瘠的、不认识你的土壤里，它长不出你想要的果子。但只要土壤够肥沃、够"你"，哪怕是一颗很普通的种子，也能长出带着你独特味道的东西来。

所以这一步千万别跳过。 在你研究任何 Skill、任何 Prompt、任何工作流之前，先花时间老老实实写一份"我是谁"。这份文件越丰富，AI 就越像你，你后面长出来的 Skills 也就越是你的。

光说"土壤"和"种子"太虚，我拿我自己跑了大半年的小红书业务，给你看点实的。

先说个反常识的事：我搭 Skills，不是从写 Skill 开始的，是从搭知识库开始的。

这话什么意思？很多人一上来就想着"我要做个写笔记的 Skill"，但其实写笔记这个动作，背后得有一整套东西托着它。没有这套东西，Skill 就是空中楼阁。

我的整套小红书 Skills 体系，是按这个顺序长出来的：

```
【产品知识库】
     ↓
【选题库 Skill】
     ↓
【选题创作规范 Skill】
     ↓
【核心关键词库】
     ↓
【关键词使用指南 Skill】
```

一个一个说。

6.1 一切的地基：产品知识库

最底下这一层，是我那套产品本身，Leo's Design Notes。12 大模块、65+ 章节、25 个品牌拆解、80+ 正反对比案例，全是我 10 年设计经验和 AI 深度共创出来的东西。

这一层为什么重要？因为我后面所有的笔记、所有的选题，都是从这套产品里"切"出来的。产品解决的是中高级设计师"做不出高级感"的痛点，那我的选题、我的内容，全部都是围绕这个痛点在打。

这套知识库本身，就是我隐性知识最大的一块沉淀。 它不是 Skill，但它是所有 Skill 的弹药库，AI 写任何一篇笔记，都是从这个库里调料。

这就是为什么别人抄不走。他可以抄我的笔记格式、抄我的标题套路，但他抄不走我这套花了大半年、几十万字搭出来的知识库。没有这个底座，再好的 Skill 也跑不出我的东西。

6.2 选题 Skill：解决「今天写什么」

知识库搭好了，第一个长出来的 Skill 是选题。

我手上有个 500+ 的大型选题库，全是围绕产品人群的痛点设计的，按紧迫程度排成 P0-P1-P2-P3-P4。最痛的痛点排最前面，依次往下。选题 Skill 干的活儿，就是从这个库里按规则帮我挑。

我现在的操作，简单到只剩一句话：

```
我：规划第 3 轮选题

AI：已读取选题库，按 3+7 配比筛出 10 个选题
    每个都带：搜索词 / 痛点场景 / 买点描述
    1. ……
    2. ……
    （共 10 条）

我：确认            ← 或者：换掉第 4 个
```

这个 Skill 背后封装的是什么？是我对"哪个痛点更痛、应该先打哪个"的判断。这个配比为什么是 3+7 而不是 5+5？是我跑了几百篇笔记之后摸出来的手感。这个手感，就是别人复制不走的隐性知识。

不过丑话说在前面，选题这事儿，AI 筛归筛，人不能撒手。

我一轮选题 10 篇，30% 热点流量型，70% 干货转化型。AI 按这个比例筛没问题，但筛完我必须自己过一遍，主要盯两件事。

一个是比例别跑偏。热点的塞多了，流量是有了，不转化，白热闹；干货塞多了，转化是稳，但没新人进来。这个度得人把着。

另一个是看选题有没有跑出我产品的框架目录。AI 有时候会夹带一些私货进来，看着也像个选题，但跟我产品八竿子打不着，或者痛点软绵绵的没劲儿。这种发出去就是浪费一次曝光，必须人工给它毙掉。

还有个小细节，很多人想不到：我会让 Codex 隔三差五清理一遍那些过期的数据报告、文件迁移报告。这些破文件堆在系统里，AI 读指令的时候容易被带偏，拿着旧数据当新依据用。屋子得常打扫，干活才利索。

6.3 笔记创作 Skill：把「我的写法」固定下来

选题定了，接下来是写，这是我最花心思的一个 Skill。

一篇笔记在我这儿不是一段文字，是一整套东西：6 张封面图文 + 一篇正文 + 10 个话题标签 + 评论区文案。每一块都有我自己的硬规矩，全写进了这个 Skill 里。

光说几条我自己的标准你就能感觉到密度：

•

标题：≤20 字，必须命中用户搜索词 + 戳中具体痛点，绝对不许用卖点思维写

•

正文：≤1000 字，必须是买点思维（讲用户得到什么，不讲产品有什么）

•

痛点归因：只许外部归因，绝对禁止说"你审美差""你不够努力"这种内部归因的话

•

三层定位：每篇都要先想清楚"找人、找情绪、找场景"

•

结尾钩子：软性承接到产品，让用户意识到要系统解决就得看完整版

这里面我还专门塞了一套反 AI 味的规则。因为 AI 写东西有几个改不掉的毛病：爱用连续的短句、爱打破折号、爱分一大堆小段落。这些东西一出现，AI 味就藏不住了。所以我在 Skill 里明确写死：不许连续短句、不许用破折号、不许过度分段。就这几条规则，让 AI 写出来的东西，AI 味降了一大半。

你看，这个 Skill 里的每一条，都是我自己写作的真实标准。 不是从哪个模板抄来的，是我做这个业务一篇篇磨出来的。把这些东西封装进去，AI 写出来的笔记才像"简哥写的"，而不是"某个 AI 写的"。

6.4 关键词布局 Skill：把流量埋进去

笔记写完还没结束，还得布局搜索关键词。

这是很多人忽略的一块，但对我来说是战略级的。小红书有两种流量，系统推荐的流量量大但不精准，用户搜索的流量量小但极其精准，我两种都要吃。

所以我专门搭了一个核心搜索词库，又设计了一套关键词使用指南，告诉 AI 每篇笔记写完后，在封面、标题、正文开头中间结尾、话题、评论区这些位置，分别该埋什么词、埋多大密度。这个使用指南就像个"调度指挥官"，把整个词库合理地分配到一篇笔记的各个角落。

有了这套东西，哪怕我哪天不发笔记，靠着长期预埋的核心关键词，也会有源源不断的搜索流量进来下单，这就是预埋的威力。这套词库还是活的，越用越新，我每周会把店铺后台那些有成交、高点击的关键词捞出来补进词库里，再按转化数据排优先级。

6.5 全部串起来：一句话跑通一条流水线

把上面这几个 Skill 串起来，就是我现在的生产线。

我们做一套小红书笔记，指的是 1 篇文案 + 6 张封面图。每篇都走固定路线：选题 → 写文案 → 配人设图 → 做封面 → 截图出图 → 发布。

我现在一句话就能启动整条流水线。下完指令，可以出去溜达一圈再回来，AI 还在帮我干活，基本处于半自动状态，出了错它会自己打回、自己修正。

可能有人要问了，这套东西跑出来的初稿，到底能不能直接用？

说个实在的数：我一般也就改 10% 不到。

改哪呢？就两个地方，P1 封面和正文标题。别的地方真不怎么碰，就这两块，每篇我都得自己上手抠。

为啥？因为这两块管的是点击率，是第一道关。AI 写出来的标题，你按结构看都没毛病，痛点也抛了、方案也给了、数字也带了、公式感也有了，全对。但全对没用，你还得拿起来品一品，这句话有没有情绪，能不能把用户的情绪给搅起来。

同样意思的一句话，带不带那点情绪，点击率能差出一倍去。这个东西 AI 是算不出来的，它只能把"对"做出来，"感觉"得人来加。

剩下的我就扫一眼正文结尾，看转化引导顺不顺，一般没啥问题，有模板托底。关键词、话题、评论区文案这些，跑出来啥样就是啥样，直接用。

所以现在的分工就是：AI 把 90% 的对做完，我往剩下 10% 里塞感觉。 挺舒服的。

我每周的节奏大概是这样：周一调研关键词和竞品，周二生成文案，周三做封面，周四同步飞书，周末逐篇改完发布。4 个账号，每周稳定出内容，我的核心精力只花在最后的人工润色上，其他全交给系统跑。

说到底，这套东西的本质就是：把我这 10 年的经验，变成了一套可以反复调用的工具，而不是每次都从零开始跟 AI 解释一遍我是谁、我要什么。 这里面没有一个 Skill 是我从网上下载的，全是从我自己的业务土壤里，一个一个长出来的。

小红书这套讲完，你可能会觉得，那 Skills 不就是搭知识库、做选题、写内容这套路子吗？换个业务照搬就行了呗。

还真不是。我用我的公众号给你看看，同一个人、同一套底层系统，换一个业务，长出来的 Skills 完全是另一副样子。

我的公众号写的是个人成长、创业复盘、AI 方法论这类东西，跟小红书那套设计干货完全两回事。底层用的还是 Obsidian + Claude Code / Codex 这套系统，但喂给它的东西，和长出来的 Skills，都不一样。

7.1 同一套系统，喂的是另一份「我是谁」

这里有个细节我必须单独拎出来讲，因为它特别能说明问题。

我的 Obsidian 里其实有两份「我是谁」的文件：

```
Obsidian Vault/
├── 人物档案.md          # 真实的简哥 → 公众号 / 社群 / 个人品牌
│
└── Leo-IP/
    └── IP定位.md         # 博主 Leo 人设 → 小红书
```

一份是 Leo 的 IP 定位文件，给小红书用的，那里面的"我"是博主 Leo，是那个专门讲设计审美、服务中高级设计师的人设。另一份才是我前面给你看的那个人物档案，写的是真实的简哥，35 岁、8 次创业、屡败屡战、骨子里热血，这份给公众号、朋友圈、社群分享用。

我专门在档案里写了一条规矩：小红书内容不许调用这份真实档案，公众号、个人品牌的内容才用它。

为什么要分两份？因为这是两个不同的场景，两个不同的"我"。

小红书上读者要的是干货、是解决方案，他们想看的是"设计专家 Leo"。公众号上读者想看的是一个真实的人，是简哥这一路怎么摔过来又怎么爬起来的。如果我把这两份档案混着用，AI 写出来的东西就会精神分裂，小红书笔记里冒出我创业失败的苦情，公众号文章里又全是冷冰冰的设计术语，这就会感觉非常奇怪了。

所以你看，「让 AI 认识你」这件事，还得分清楚是哪个场景里的你。 同一个人，在不同业务里有不同的身份，要用不同的角色和不同的身份，去做不同项目里面的工作。这就是为什么我说， [CLAUDE.md](http://claude.md/) 这件事，越做越深，越做越细。

7.2 公众号的当家 Skill：一鱼多吃

小红书的核心 Skill 是"选题 + 创作 + 关键词"，到了公众号，当家的变成了另一个，我管它叫"一鱼多吃"。

这个 Skill 解决的是一个我特别头疼的问题：我有大量的原始语料，但它们都散着、乱着、用不起来。

比如我经常会有一些长语音，自己开车路上口述的想法、跟圈友 1 对 1 交流的录音、社群里的一段分享。这些东西信息密度很高，全是我真实的思考，但一直躺在那儿吃灰，因为整理它们太费劲了。

一鱼多吃 Skill 干的就是这件事，我把一段录音稿丢给它，它直接帮我加工成好几样东西：

•

一篇公众号长文的初稿

•

几条朋友圈文案

•

一篇小红书笔记的角度

•

外加把里面有传播力的金句、观点全部提炼出来

一场一两个小时的直播或者分享，语料经过这么一加工，够我用上大半个月。

这个 Skill 为什么有效？因为它封装的还是我的东西。它知道我说话的语气是"直接、真诚、不装、带点江湖气"，知道我哪些观点是反复在讲的内核，知道我的金句一般长什么样。换个人的一鱼多吃 Skill 拿过来，加工出来的就不是简哥的味道了。

7.3 长文 Skill：把「反 AI 味」做到极致

公众号还有一个绕不开的 Skill，就是写长文。

写长文最怕什么？最怕 AI 味。一篇个人成长文，读者一眼看出来是 AI 写的，那点真诚和信任瞬间就没了。所以我的长文 Skill 里，反 AI 味的规则比小红书还要严。

里面写死了好几条我自己的写作铁律：不许有连续的短句、不许滥用破折号、不许分太多的小段落、不许出现"首先其次最后"这种八股结构、不许讲正确的废话。这些规则全是我自己写东西时的真实习惯，一条条抠出来塞进去的。

你现在读的这篇文章，本质上就是在这个长文 Skill 的辅助下，我们一段一段磨出来的。它知道我是谁、知道我的经历、知道我的语气、知道我讨厌什么样的表达。所以它能给我递上来的，是已经有七八分"简哥味"的初稿，我再在上面改、调、加料，效率比从零写高太多了。

7.4 这两个案例放一起，结论就清楚了

把小红书和公众号这两个案例摆在一起看，一个特别重要的结论就浮出来了：Skills 不是通用的。它长成什么样，完全取决于你的业务任务是什么、你这个人是谁。

同样是我，同样用 Obsidian + Claude Code，因为小红书要的是高频、标准化、搜索导向的短笔记，它就长出了"选题 + 创作 + 关键词"这套 Skill。因为公众号要的是有真实感、有观点、有传播力的长内容，它就长出了"一鱼多吃 + 长文"这套 Skill。土壤不同（两份不同的"我是谁"），任务不同（短笔记 vs 长文章），种子自然长成不同的样子。

这也是为什么我一直说，别人的 Skills 你真的抄不走。 不是技术抄不走，是那片土壤、那个具体的人、那个具体的业务场景，抄不走。

说了这么多，你可能已经心痒了，想自己动手做一个。

好事。但动手之前，我先把话说在前面：原创一个 Skill，根本不需要你会写代码。 真正难的不是技术，是想清楚一件事，你的重复任务到底是什么。

我自己摸索下来，就三步。

8.1 第一步：找出你的高频重复任务

打开你的记事本，把你每天、每周都在重复做的事情列出来。

注意两个关键词，高频和重复。偶尔做一次的不算，每次做法都不一样的也不算。要找的是那种"我这周做了、下周还得做、做的方式还都差不多"的事。

比如我做小红书笔记，那我的高频任务就是审稿；我是做客服的，高频任务就是回各种咨询；我是做知识付费的，高频任务就是写笔记、做选题。把这些列出来，越高频的越往前排。

这一步看着简单，其实很多人卡在这儿，因为他们觉得自己每天做的事"没什么技术含量""不值得做成 Skill"。这个心态后面我专门说，先记住一句话：越是你闭着眼睛都能做的高频活儿，越值得 Skill 化。

8.2 第二步：把你的判断标准掏出来

这是最关键、也是最难的一步。

找到那个高频任务之后，你得问自己：我做这件事的时候，脑子里到底在按什么标准判断？

还是拿审稿举例。你审一篇稿子，肯定不是瞎看的，你心里有几个固定的点：这个标题行不行、这个开头能不能抓住人、这段逻辑顺不顺、这里有没有硬伤。这几个点，就是你的判断标准。

把它们一条一条掏出来，写清楚。这一步就是在做我前面说的那件事，把你脑子里的隐性知识，变成可以描述的显性规则。

这一步做得越细，你的 Skill 就越像你。我那个笔记创作 Skill 里塞了几十条标准，标题多少字、用什么思维、归因方式、哪些词不能用，全是我掏出来的判断标准，掏得越干净，AI 替我干活就越省心。

我教你个小窍门：如果你一时想不清楚自己的标准是什么，就去回想"我上次否掉一个东西，是因为它哪里不对"。你否定的理由，往往就是你最真实的判断标准。

8.3 第三步：让 AI 做出第一版，然后往死里改

标准掏出来之后，剩下的就交给 AI 了。

打开 Claude Code 或者你用的任何 AI，把你的任务和判断标准一股脑告诉它，让它帮你做出第一版 Skill。第一版肯定不完美，没关系，你拿你的真实任务去测试它，跑一遍看看输出对不对。

不对就改。是规则没写清楚？补上。是 AI 理解偏了？说明白。是漏了某个标准？加进去。就这么一轮一轮地测、一轮一轮地改，直到它输出的东西让你满意为止。

这里有个我自己踩过的最大的坑，一定要提醒你：别想着让一个 Skill 把所有事都干了。

我刚开始就犯过这个错，想做一个"全能 Skill"，让它一次性帮我找对标、定选题、写文案、做配图，全都包圆。结果呢？做出来的东西特别烂。永远没有一步到位的东西。越想走得快，其实回过头来走得越是慢。慢慢走、慢慢调试，才是最快的捷径。

后来才想明白为什么。因为你让一个 Skill 一次干三件事，它就要做三次独立判断，每一次判断都可能跑偏你的标准。三次连乘下来，跑出来的东西早就不知道飞哪儿去了。

正确的做法是拆开。 选题 Skill 只管选题，创作 Skill 只管创作，关键词 Skill 只管布局关键词。每个 Skill 只干一件事，只做一次判断，结果就稳得多，也好维护得多。

记住六个字：专而精，不贪大。 这是我交的学费里最值钱的一条。

最后再补充一个大家肯定会关心的问题：Skill 做出来之后，怎么维护？

我的真实情况是：骨架基本不动，微调实时在做。

为什么骨架不能随便动？因为我的每个 Skill 之间都是强关联的。一个 Skill 一变，整篇笔记的结构都要跟着变，再往上推算，每一个关联的 Skill 都得相应调整，牵一发动全身。所以 Skill 的主体框架，定下来之后我轻易不碰。

但小的微调是一直在发生的。根据笔记数据，新增一些小规范、收录一些新出现的关键词、给封面 P1-P5 的设计规范里加几个引导词、调整一下字体大小的规则，这类小修小补每周都有，改起来也就几分钟的事。

什么时候才会动大的？我给自己定的信号是：如果发现笔记的封面点击率持续走低，或者阅读量出现 2-3 个月的下降趋势，那说明不是某篇笔记的问题，是底层规范过时了。这时候我才会坐下来，重新修改封面设计规范和关键词布局策略。

一句话总结我的维护逻辑：数据不报警，只做小保养；数据报警了，才动大手术。

写到这儿，方法层面的东西基本说完了。但我觉得最后这两句，可能比前面所有的方法都重要。

因为这大半年我接触了不少想用 AI 做事的圈友，发现真正卡住他们的，往往不是技术，是两个心结。我自己也曾经被这两个心结困住过，今天一并说破。

太多人有这个想法了。

觉得自己每天做的事平平无奇，谁都会，没什么了不起，哪有什么值得封装成 Skill 的东西。我太理解这种感觉了，因为一件事你做了十年，熟到闭着眼睛都能做，你就会真的以为它不值钱。

但我想跟你说，恰恰相反。

你那些"看一眼就知道行不行"的判断力、你那套自己都没意识到的操作流程、你踩过无数坑才换来的标准，这些东西对你来说习以为常，但对别人、尤其是对 AI 来说，是极其珍贵的。

我做 567 篇笔记才摸出来的标题手感，我拆解 25 个品牌才沉淀的审美判断，在我自己看来不过是"日常"，但正是这些"日常"，让我的 Skill 长出了别人复制不走的样子。

Skill 的价值从来不来自技术有多复杂，来自你的隐性知识有多稠密。 你每天在做的那件事，对你自己不值钱，可一旦把它固化成 Skill，它就值钱了，因为你把一件只能由你本人来做的事，变成了一个可以独立运转的系统。所以别瞧不起自己的活儿，你那点"习以为常"，才是金子。

还有人是另一种纠结，觉得自己那套东西太土、太朴素，不够高级，不好意思拿出来。

我以前也这么想，后来想通了一个道理：专业是相对的，不是绝对的。

60 分的人可以教 40 分的人，40 分的人可以教 20 分的人。你不需要做到行业顶尖，你只需要比你的用户、比那个还没追上你的人，多走那么几步就够了。

我做设计 10 年，在真正的设计大神面前我不算什么。但对于那些还在为"做不出高级感"发愁的 3-5 年设计师来说，我趟过的路、我总结的标准，对他们就是有用的。你的方法论不需要惊天动地，它只需要真实、来自你的实战、能解决一群具体的人的具体问题，这就够了，这就值钱了。

回到开头那个问题：为什么别人的 Skills 到你手里就废了？

现在你应该有答案了。不是 AI 不够强，是 AI 根本不认识你。不是 Skill 不够好，是它不是从你的土壤里长出来的。

我用 AI 这一路，最大的收获不是学会了多少 Prompt 技巧、收藏了多少 Skill 包，而是终于想明白了一件事：

AI 再强，它也只是放大器。它放大的，是你这个人本身。

你是谁、你有多少真东西、你愿不愿意把自己这个人完整地交给它，决定了它能帮你长出多厉害的东西。所以别再到处去搬别人的 Skill 了，先回过头，老老实实写一份"我是谁"，把你这些年攒下的真东西，一点一点喂给它。

种子谁家的都行，土壤必须是你自己的。

我也还在路上，远没到能单干的程度，但方向我是认准了。如果这篇东西能让你在搭自己系统的时候少走一段弯路，那我就觉得蛮欣慰的了。

Let's go!一起加油，一起生财有术！
